# Contributing to Locked - Offline Password Manager

Thank you for considering contributing to Locked! Contributions are welcome from everyone. By participating in this project, you agree to abide by the following guidelines.

## How to Contribute

1. Fork the repository and clone it locally:
   ```
   git clone https://github.com/your_username/Locked.git
   cd Locked
   ```

2. Create a new branch for your feature or bug fix:
   ```
   git checkout -b feature/new-feature
   ```

3. Make your changes and commit them with descriptive commit messages:
   ```
   git commit -am 'Add new feature: Description of your changes'
   ```

4. Push your changes to your fork:
   ```
   git push origin feature/new-feature
   ```

5. Submit a pull request:
   - Go to the [Pull Requests](https://github.com/your_username/Locked/pulls) tab of the Locked repository.
   - Click on "New Pull Request".
   - Provide a brief summary of your changes in the title and description fields.
   - Submit the pull request.

## Code Style

- Follow Python's [PEP 8](https://www.python.org/dev/peps/pep-0008/) style guide.
- Use meaningful variable and function names.
- Maintain consistent indentation (4 spaces).

## Testing

- Ensure your changes do not break existing functionality.
- Write tests for new features or changes wherever possible.
- Run existing tests to verify your changes.

## Security

- Do not introduce security vulnerabilities.
- Avoid hardcoding sensitive information.
- Use safe cryptographic practices for handling passwords and sensitive data.
